## 1. DESCRIPCIÓN GENERAL DEL PROYECTO

### 1.1 ¿Qué es ConstrucTrack LogBook?

**ConstrucTrack LogBook** es una aplicación de escritorio basada en **JavaFX** diseñada para la gestión integral de:

- **Proyectos de construcción**: Registro de obras y actividades
- **Bitácoras técnicas**: Documentación diaria de progreso
- **Evidencia de desempeño**: Almacenamiento de fotos y documentos
- **Usuarios y permisos**: Control de acceso y autenticación

### 1.2 Características Técnicas

| Aspecto | Especificación |
|--------|---|
| **Lenguaje** | Java 17 LTS |
| **Interfaz** | JavaFX 21.0.2 |
| **Base de Datos** | MySQL 8.0+ (teórico) |
| **Build Tool** | Apache Maven 3.8+ |
| **Autenticación** | Usuario/Contraseña con BCrypt |
| **Exportación** | PDF de reportes |
| **Arquitectura** | 21 clases Java + 6 vistas FXML |

### 1.3 Flujo de la Aplicación

```
Usuario → Autenticación (LoginController) 
       → Dashboard (DashboardController)
       → Gestión de Proyectos (ProyectoFormController)
       → Gestión de Bitácoras (BitacoraFormController)
       → Carga de Evidencia (RegistroController)
       → Exportación PDF
```

## 4. PARTE 2: ARQUITECTURA UBUNTU + MYSQL + APACHE (ESTRUCTURA TEÓRICA)

**Nota:** Este es el despliegue real en servidor Linux, documentado como referencia académica.

### 4.1 Arquitectura Completa

```
┌──────────────────────────────────────────────┐
│         Cliente (Navegador)                   │
└──────────────────────────────────────────────┘
                    ↓ HTTP:80
┌──────────────────────────────────────────────┐
│        Apache 2.4 (Proxy)                    │
│   - ProxyPass → localhost:8080               │
│   - Compresión, logs, SSL                    │
└──────────────────────────────────────────────┘
                    ↓ HTTP:8080
┌──────────────────────────────────────────────┐
│     Aplicación Java (Port 8080)              │
│   - JDK 17: /usr/lib/jvm/java-17-*          │
│   - JAR: /opt/constructrack/target/         │
│   - Logs: /var/log/constructrack/           │
└──────────────────────────────────────────────┘
                    ↓ TCP:3306
┌──────────────────────────────────────────────┐
│     MySQL 8.0 (Database)                     │
│   - Puerto: 3306                             │
│   - BD: constructrack_logbook               │
│   - Usuario: constructrack                  │
└──────────────────────────────────────────────┘
```

### 4.2 Configuración en Ubuntu (Pasos Teóricos)

**PASO 1: Sistema Base**

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y openjdk-17-jdk maven mysql-server apache2
```

**PASO 2: Configurar MySQL**

```bash
sudo mysql -u root
> ALTER USER 'root'@'localhost' IDENTIFIED BY 'root123';
> CREATE DATABASE constructrack_logbook;
> CREATE USER 'constructrack'@'localhost' IDENTIFIED BY 'constructrack123';
> GRANT ALL PRIVILEGES ON constructrack_logbook.* TO 'constructrack'@'localhost';
> FLUSH PRIVILEGES;
```

**PASO 3: Configurar Apache Proxy**

```bash
sudo a2enmod proxy proxy_http
sudo nano /etc/apache2/sites-available/constructrack.conf
```

Contenido:
```apache
<VirtualHost *:80>
    ProxyPreserveHost On
    ProxyPass / http://localhost:8080/
    ProxyPassReverse / http://localhost:8080/
</VirtualHost>
```

**PASO 4: Compilar y Ejecutar como Servicio**

```bash
cd /opt/constructrack
mvn clean package -DskipTests
sudo systemctl start constructrack
```
