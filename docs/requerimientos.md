# Requerimientos del Sistema

## Funcionales
- [ ] RF1: Autenticacion y gestion de usuarios.
- [ ] RF2: Gestion de proyectos (crear, editar, eliminar, listar, asignar estado).
- [ ] RF3: Bitacora diaria por proyecto (crear, editar, eliminar, listar, exportar PDF).
- [ ] RF4: Gestion de evidencias/archivos asociados.
- [ ] RF5: Reportes y metricas clave.

## No funcionales
- [ ] RNF1: Seguridad de credenciales (hash de passwords).
- [ ] RNF2: Rendimiento (respuesta < 2s en consultas tipicas).
- [ ] RNF3: Disponibilidad (segun SLA acordado).
- [ ] RNF4: Portabilidad (Java 17+, JavaFX).

## Reglas y consideraciones
- [ ] Accesos por rol (admin/usuario).
- [ ] Validaciones basicas de formularios.
- [ ] Integridad referencial en BD (proyecto-bitacora-usuario).

## Estado de aprobacion
- [ ] Revisado por cliente/PO
- [ ] Aprobado (fecha y firma)
